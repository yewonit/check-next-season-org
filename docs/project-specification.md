## 1. 프로젝트 개요

### 1.1 프로젝트 목적

- 2026년 청년회 순배치 결과를 온라인으로 확인할 수 있는 웹 서비스 제공
- 재미있고 감동적인 UX를 통해 새로운 순 배치에 대한 기대감 형성
- 동시 접속 트래픽 환경에서 안정적인 서비스 제공

### 1.2 조직 구조

```
청년회
├── 숫자 국 (1국, 2국, 3국...)
│   ├── 그룹 (그룹장 이름)
│   │   └── 순 (순장 이름)
│   └── 올레이션국 (외국인)
│       ├── A그룹, B그룹, C그룹...
│       └── 순 (순장 이름)
```

### 1.3 예상 사용자 환경

- **예상 동시 접속자**: 600~900명
- **피크 타임**: 공개 시점(예: 오후 6시) 일제히 트래픽 집중
- **주요 디바이스**: 모바일 (스마트폰)
- **사용 환경**: PWA 또는 모바일 웹

## 2. 사용자 플로우

### 2.1 일반 사용자 플로우 (한국인)

```
[메인 페이지]
  ↓
이름 입력 및 검색
  ↓
[중복 확인]
├─ 중복 없음 → [애니메이션 페이지]
└─ 중복 있음 → [중복자 선택 모달] → [애니메이션 페이지]
  ↓
[결과 페이지]
```

**총 플로우 단계**: 3~4단계 (중복 여부에 따라)

### 2.2 올레이션(외국인) 플로우

```
[메인 페이지]
  ↓
"올레이션국이라면 여기 클릭" 버튼
  ↓
[올레이션 전용 페이지 - 그룹 선택]
  ↓
그룹 선택 (A그룹, B그룹, C그룹...)
  ↓
[올레이션 명단 페이지]
  ↓
본인 이름 선택
  ↓
[애니메이션 페이지]
  ↓
[결과 페이지]
```

**총 플로우 단계**: 4단계

## 3. 화면 상세 정의

### 3.1 메인 페이지 (이름 입력)

#### 화면 구성

**헤더 영역**
- 배지: "2026년 청년회"
- 메인 타이틀: "순배치 확인"
- 서브 텍스트: "이름을 입력하여 나의 순을 확인하세요"

**입력 영역**
- 이름 입력 필드
  - 플레이스홀더: "이름을 입력하세요 (예: 홍길동)"
  - 입력 규칙: 성+이름 필수 (Full name)
- 검색 버튼 (입력 필드 우측에 아이콘 버튼)

**보조 버튼**
- "올레이션국이라면 여기를 클릭하세요 →" 텍스트 버튼

#### 주요 기능

- 이름 입력 시 실시간 유효성 검사
- 엔터키로 검색 실행 가능
- 검색 버튼 클릭 시:
  - 해당 이름 검색
  - 중복 여부 확인
  - 중복 없음 → 애니메이션 페이지로 이동
  - 중복 있음 → 바텀시트 모달 표시

#### 디자인 방향

- 깔끔하고 미니멀한 디자인
- 그라데이션 배경 (blue-purple 계열)
- 큰 입력 필드로 터치 친화적 설계

---

### 3.2 중복자 선택 모달 (바텀시트)

#### 동작 방식

- **트리거**: 입력한 이름이 2명 이상 존재할 경우
- **애니메이션**: 하단에서 슬라이드 업 (지도 앱 스타일)
- **배경**: 딤 처리 (반투명 검은색)
- **높이**: 화면의 최대 75%

#### 화면 구성

**드래그 핸들**
- 상단 중앙에 가로 막대 표시
- 드래그로 모달 닫기 가능 (선택 구현)

**헤더**
- 타이틀: "동일한 이름이 있습니다"
- 서브 텍스트: "본인의 정보를 선택해주세요"
- 닫기 버튼 (X 아이콘)

**리스트 영역**
- 중복된 사람 수만큼 카드 표시
- 각 카드 정보:
  - 이름 (크고 굵게)
  - 기수 (뱃지 형태)
  - 고유번호 (라벨: 값 형식)
  - 전화번호 (라벨: 값 형식)
- 스크롤 가능

**카드 디자인**
- 흰색 배경에 회색 테두리
- 호버/터치 시 파란색 테두리로 변경
- 오른쪽 화살표 아이콘으로 선택 가능함을 표시
- 그림자 효과로 입체감

#### 인터랙션

- 카드 클릭 시 해당 사용자 선택
- 모달 닫힘과 동시에 애니메이션 페이지로 이동
- 배경 또는 닫기 버튼 클릭 시 모달만 닫힘

---

### 3.3 올레이션 전용 페이지 - 그룹 선택

#### 화면 구성

- **타이틀**: "2025년 나는 어디 그룹이었나요?"
- **설명 텍스트**: "작년에 속했던 그룹을 선택해주세요"

**그룹 선택 영역**
- A그룹, B그룹, C그룹, D그룹... (실제 그룹 수만큼)
- 큰 버튼 형태로 표시
- 각 버튼에 그룹명 표시

#### 인터랙션

- 그룹 버튼 클릭 → 해당 그룹의 명단 페이지로 이동

#### 디자인 가이드

- 버튼은 2열 그리드 또는 1열 리스트
- 터치 영역 충분히 확보
- 선택 시 시각적 피드백

---

### 3.4 올레이션 명단 페이지

#### 화면 구성

**헤더**
- 선택한 그룹명 표시 (예: "A그룹 명단")
- 뒤로가기 버튼

**명단 리스트**
- 해당 그룹 소속 인원 전체 표시
- 롤업 애니메이션으로 등장
- 이름 리스트 (가나다순 또는 알파벳순)

#### 인터랙션

- 본인 이름 클릭 → 애니메이션 페이지로 이동

#### 특이사항

- 올레이션국 총 인원: 약 200명
- 그룹별로 분산하여 스크롤 부담 감소
- 한글 이름으로 표시 (DB 기준)

---

### 3.5 애니메이션 페이지

#### 컨셉

"선물을 열어보는 설렘"

#### 화면 구성

- **배경**: 심플한 단색 또는 그라데이션
- **중앙 요소**: 선물 박스 아이콘
- **안내 텍스트**: "2026년 나의 그룹과 순을 확인하세요" 또는 "탭하여 선물을 열어보세요"

#### 인터랙션 시퀀스

1. **초기 상태**: 작은 선물 박스 표시
2. **탭 액션**: 사용자가 선물 박스를 탭
3. **점진적 확대**: 탭할 때마다 선물 박스 크기 증가
4. **폭발 효과**: 일정 크기 도달 시 (또는 3~4회 탭 후) "펑!" 하고 터짐
5. **결과 전환**: 파티클 효과와 함께 결과 페이지로 자연스럽게 전환

#### 세부 요구사항

- **최소 탭 횟수**: 3~4회
- **피드백**: 탭할 때마다 시각/촉각 피드백 (진동, 애니메이션)
- **진행 표시**: 선물 박스 크기 변화로 진행도 암시
- **폭발 애니메이션**: 축하 분위기의 파티클/컨페티 효과

#### 제외된 대안들

- ❌ **복권 긁기**: 종교적 민감성 우려
- ❌ **카드 뒤집기**: 인터랙션이 다소 단조로움
- ❌ **흔들기**: PWA 환경에서 센서 접근 제약

---

### 3.6 결과 페이지

#### 화면 구성

**상단 영역**
- 축하 메시지 또는 인사말
- 연도 표시: "2026년"

**핵심 정보 영역** (블러 해제 후 또는 카드 형태)
- "나의 그룹": [그룹장 이름]그룹
- "나의 순": [순장 이름]순
- 내가 속한 순번 (예: 1순, 2순...)

**순원 목록 영역**
- 토글/버튼: "우리 순원 전체 보기" 또는 "다른 순원 살펴보기"
- 클릭 시 같은 순에 속한 전체 인원 목록 표시
- 리스트 형태로 이름 나열

#### 인터랙션

- 애니메이션 페이지에서 전환 시 핵심 정보만 먼저 표시
- "전체 보기" 버튼 클릭 시 순원 목록 펼침
- 공유 기능 (선택 사항)
- 홈으로 돌아가기 버튼

#### 디자인 가이드

- 축하 분위기의 밝은 색상
- 카드 기반 레이아웃으로 정보 계층 구분
- 순원 목록은 접었다 펼칠 수 있는 아코디언 형태

## 4. 데이터 요구사항

### 4.1 사용자 입력 데이터

- **이름**: 성+이름 (Full name, 필수)
- **입력 방식**: 텍스트 입력 필드

### 4.2 데이터베이스 필수 정보

각 청년회원에 대해 다음 정보 필요:

| 필드명 | 설명 | 비고 |
|--------|------|------|
| 이름 | 성+이름 | 검색 키 |
| 기수 | 23기, 24기, 25기 등 | 중복자 식별용 |
| 고유번호 | YA-YYYY-XXXX 형식 | 중복자 식별용 |
| 전화번호 | 010-XXXX-XXXX | 중복자 식별용 (정확도 낮을 수 있음) |
| 국 | 숫자국 or 올레이션국 | 플로우 분기용 |
| 그룹 | 그룹장 이름 | 2025년 소속 그룹 (올레이션만) |
| 2026년 그룹 | 그룹장 이름 | 배치 결과 |
| 2026년 순 | 순장 이름 | 배치 결과 |
| 순원 목록 | 같은 순 소속 인원 | 배열 또는 관계 |

### 4.3 데이터 정합성

- **사전 작업 필요**: 기수, 전화번호 미입력 데이터 정비
- **올레이션 데이터**: 한글 이름 표기 통일 (일부 영어 표기 혼재)
- **중복 처리**: 동명이인 최대 3~5명 예상

## 5. 주요 기획 결정사항

### 5.1 입력 방식 결정

**선택**: 이름만 입력 (성+이름)

**장점**:
- UI 단순화, 사용자 진입장벽 낮음
- 기수를 모르는 청년들 고려
- 중복 해결: 바텀시트 모달로 추가 정보 제공하여 선택

**제외**: 기수 입력 추가

**이유**:
- UI 복잡도 증가
- 본인 기수를 정확히 모르는 경우 존재
- 전화번호 데이터 정확도 문제

---

### 5.2 올레이션 별도 플로우

**배경**
- 올레이션국 총 인원: 약 200명
- 이름 검색 시 스크롤 부담 과다
- 한글 표기 미숙으로 검색 어려움

**해결책**
- 그룹 기반 1차 필터링
- 작년 소속 그룹 선택 → 해당 그룹 명단만 표시
- 그룹당 인원 수 감소로 스크롤 편의성 향상

---

### 5.3 인터랙션 디자인 결정

**최종 선택**: 선물 박스 탭 액션

**선정 이유**:
- 종교적으로 중립적이고 긍정적인 이미지
- "선물"이라는 은유가 새 순 배치의 의미와 부합
- 반복 탭으로 참여도 높임
- 구현 가능성 확보

**비교 검토된 대안들**

| 대안 | 장점 | 단점 | 결과 |
|------|------|------|------|
| 복권 긁기 | 직관적, 재미 | 종교적 민감성 (도박 연상) | ❌ 제외 |
| 카드 뒤집기 | 깔끔한 애니메이션 | 단조로움, 차별화 부족 | ❌ 제외 |
| 흔들기 | 물리적 인터랙션 재미 | PWA에서 센서 접근 제약 | ❌ 제외 |
| 지우기/블러 제거 | 심플함 | 수동적, 임팩트 약함 | ❌ 제외 |
| 선물 박스 탭 | 긍정적, 참여형, 안정적 구현 | - | ✅ 선택 |

## 6. 사용자 경험 목표

### 6.1 핵심 UX 원칙

- **안정성 우선**: 첫 서비스 경험이 긍정적이어야 내년 기대감 형성
- **직관성**: 별도 설명 없이도 사용 가능한 단순한 플로우
- **감동 포인트**: 애니메이션을 통한 설렘과 축하 분위기 연출
- **접근성**: 올레이션 등 다양한 사용자 고려

### 6.2 성공 지표

- 서비스 오픈 시 다운타임 없이 안정적 운영
- 동시 접속자 600~900명 수용
- 사용자 이탈 없이 최종 결과 페이지까지 도달
- 인증샷 공유 등 자발적 바이럴 발생

---

## 7. 제약 사항 및 고려사항

### 7.1 기술적 제약

- **트래픽 집중**: 6시 공개 시점 동시 접속 600~900명
- **환경**: PWA 또는 모바일 웹 기반
- **안정성**: 백엔드 부하 시 대안(프론트 JSON) 고려

### 7.2 콘텐츠 제약

- **종교적 적절성**: 복권 등 도박 연상 요소 배제
- **축하 분위기**: 긍정적이고 따뜻한 톤앤매너 유지
- **다국적 사용자**: 올레이션국 별도 케어

### 7.3 일정 제약

- **회기 전 데이터 정비**: 기수/전화번호 누락 정보 입력

## 9. 부록

### 9.1 용어 정리

- **순**: 소그룹 단위 (순장 + 순원들)
- **그룹**: 순들의 모임 (그룹장 관리)
- **국**: 그룹들의 모임 (숫자국 or 올레이션국)
- **올레이션**: 외국인 청년들의 국

### 9.2 페이지 맵

```
1. 메인 페이지 (이름 입력)
├─→ 2-1. 중복자 선택 모달 (조건부)
└─→ 2-2. 올레이션 그룹 선택
    └─→ 2-3. 올레이션 명단
        └─→ 3. 애니메이션 페이지
            └─→ 4. 결과 페이지
```

**총 페이지 수**: 6개 (모달 포함)  
**사용자 경로별 단계 수**: 3~4단계
